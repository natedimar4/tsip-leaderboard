How this works
  • The leaderboard shows tasks completed this week only (Monday 12:00 AM – Sunday 11:59 PM Eastern).
  • Everyone resets to 0 every Monday morning. Past weeks don't carry over.
  • Click any name to see that person's week-by-week history, including last week's historical results.
  • Admins add task counts manually as people complete work — entries are filed into the week of their date, not the date they were typed in.

How the admin panel works

Weekly reset. Entries are assigned to weeks by their date, not by when you type them in. The leaderboard shows the current week (Mon 12:00 AM – Sun 11:59 PM Eastern). Everyone resets to 0 at midnight Monday automatically — there's no button to press.

Single entry. Pick a person (or type a new name), set the date the work was done, enter the task count, and Save.

Bulk paste. Pick one date for the whole paste. Then paste rows of Name <tab/space/comma> Count — one per line. The last whitespace/comma/tab token on each line is the count; everything before it is the name (so "Jared M 12" works). Preview before saving.

Editing past entries. Use the "Edit existing entries" section. You can filter by person or week, then edit or delete individual entries. Editing the date may move the entry to a different week (its weekStart recomputes).

Adding a new person. Just type their name in the single-entry form's "or new name" field, or include it in a bulk paste — they're created on first save.

Renaming someone. Use the "Rename person" tool — it updates every entry tied to that person.

Your name. Set "Your name" above; every entry you save is attributed to you. Make sure it's right before saving.

Backdating. If you enter a task with last Sunday's date, it lands in last week's bucket, not this week's. Use today's date for "this week" credit.

One-time historical import (last week)

Pulls last week's totals (week of 5/19 – 5/25) from the source Google Sheet and writes one entry per person stamped to that week. Runs only once.
